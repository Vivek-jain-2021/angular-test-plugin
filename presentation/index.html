/angular-test:generate
Auto-detects your Angular version and generates ready-to-run .spec.ts files for any component, service, pipe, directive, guard, resolver, interceptor, or NgRx file.
/angular-test:coverage
Analyses existing test files, identifies missing scenarios, and generates only the absent it() blocks — never touching tests that already pass.
/angular-test:fix
Diagnoses failing tests from static analysis or test logs, then applies surgical fixes for 18 documented error patterns — no full rewrites.
/angular-test:suite
Orchestrates the full pipeline for the entire project: reads angular.json, discovers all files, then runs generate → coverage → fix in dependency order.
📝 Command 1

/angular-test:generate

Generate complete, ready-to-run spec files — targeting 100% branch coverage

  • ✓Auto-detects Angular version (v14–v19) and applies the correct testing patterns and APIs
  • ✓Handles all file types: components (module & standalone), services, pipes, directives, guards, resolvers, interceptors, NgRx
  • ✓Version-specific APIs: signals, input(), output(), model(), @defer, resource API, zoneless
  • ✓Both happy & unhappy paths — error scenarios always tested, not just the success case
  • ✓Scope: single file, directory, or entire project — with --skip-existing to protect current work
--dry-run --skip-existing --overwrite --only=components --threshold=80 --runner=jest --ng=17
// Generated: user.service.spec.ts describe('UserService', () => { let service: UserService; let httpMock: HttpTestingController; beforeEach(() => { TestBed.configureTestingModule({ providers: [provideHttpClientTesting()] }); service = TestBed.inject(UserService); httpMock = TestBed.inject(HttpTestingController); }); it('should load users successfully', () => { service.getUsers().subscribe(users => { expect(users.length).toBe(2); }); const req = httpMock.expectOne('/api/users'); req.flush([{ id: 1 }, { id: 2 }]); }); it('should handle 404 error gracefully', () => { /* ... */ }); });
📊 Command 2

/angular-test:coverage

Analyse existing tests and fill coverage gaps — without touching what already works

  • ✓Per-file gap analysis with detailed tables: covered vs missing scenarios per method
  • ✓Never rewrites passing tests — adds only the missing it() blocks needed to hit threshold
  • ✓Configurable threshold — target 70%, 80%, or 100% coverage per your team's standard
  • ✓Report formats: table (terminal), JSON (CI), or HTML (stakeholder dashboard)
  • ✓Karma & Jest threshold configs generated automatically for CI enforcement
--threshold=80 --report=html --report=json --skip-complete --dry-run
// Gap analysis output (table mode) File: dashboard.component.ts Method Covered Missing ────────────────────────────────────── ngOnInit() ✓ — loadData() ✓ — onFilter() — error state exportCSV() — empty data auth error // Generated missing it() blocks: it('onFilter should handle error state', ...); it('exportCSV with empty data should warn', ...); it('exportCSV should handle auth error', ...);
🔧 Command 3

/angular-test:fix

Diagnose and surgically repair failing Angular tests — 18 documented error patterns

  • ✓Static analysis of spec files to identify failure patterns — no test run needed
  • ✓Log-driven mode: parse test run output with --from-log=path to target exact failures
  • ✓Never rewrites entire files — only targeted, minimal changes to failing blocks
  • ✓Group by error type for batch fixes across multiple files simultaneously
18 Auto-Fixed Error Patterns
NullInjectorError: No provider for X
Can't bind to property (missing declaration)
ExpressionChangedAfterChecked error
Marble test assertion failures
1 timer(s) still in the queue
Cannot use fakeAsync with real Promises
NG0950: Required input not specified (v16+)
Signal input assignment TypeError (v17+)
@defer block not rendering (v17+)
resource() value undefined (v18+)
// Before: failing test it('should display user name', () => { component.user = { name: 'Alice' }; fixture.detectChanges(); // ❌ TypeError: Cannot set property // 'user' of signal input }); // After: auto-fixed by /fix it('should display user name', () => { fixture.componentRef.setInput( 'user', { name: 'Alice' } ); fixture.detectChanges(); const el = fixture.nativeElement .querySelector('[data-testid="name"]'); expect(el.textContent).toContain('Alice'); });
--from-log=karma.log --dry-run --only=NullInjector --group-by=error --group-by=file

5
Fix All Failures
